About

Created by: Daniel McErlean

Balloon Pop is an arcade-like game built using the Flutter Flame engine. Try to beat your highscore each time you play, as the game gets more difficult the longer you last! Created for a final project at RIT.

Original Proposal

Requirements

  • Functional
    1. Game is useful (provides entertainment) and is easy to use
    2. Goes beyond other exercises by utilizing many aspects of Flame that we did not cover in class
    3. No input validation, though it is not applicable with this application
    4. Created custom splash screen, icon, and a unique app name
    5. Made use of shared_preferences to save and load volume preferences
    6. Multiple pages are present
    7. The app does not crash
    8. Documentation page ( right here :) )
  • Design & Interaction
    1. Graphics are pleasing and fit the aesthetic
    2. All widgets are labeled exactly for what they do
    3. Users should be able to figure out what to do, and instruction is provided anyway
    4. App is responsive for portrait devices
  • Media
    1. Images are optimized and not overly large
    2. Sounds effects and background music are present
    3. A custom font is utilized
  • Code Conventions
    1. Multiple custom dart classes are used
    2. Code is type aware, var is never used
    3. Repetive code is broken up into reusable functions/methods
    4. Variables and functions are named for what they do
    5. Comments describe exactly what each function does
    6. No print statements

Process

  1. Came up with an idea for a simple arcade game that involved popping balloons
  2. Researched Flame and different ways of laying out the app navigation
  3. Settled on overlays, and found different a Flame tutorial that explained in depth on how to make a flame game
  4. Set up the main game class, where everything is initialized and the game loop is started
  5. Wrote a class that can take in multiple types of balloons to make creating different versions easier and less redundant
  6. Created the game loop, where the balloons rise up, pop if they leave the screen, causing the player to lose a life. Popping a balloon before it leaves increases the players score.
  7. Created the overlays and set them up with different states such as the pause state when pressing the pause button.
  8. Finally, I added music and sound effects, and volume controls.

Set Backs / Glitches

  • After a while of playing, the sound effects may become delayed.
  • Music is restarted when changing the volume due to FlameAudio not allowing variables to initialize music. (could be fixed by creating the audio on the main page with AudioPlayers, but it was too late to change it)
  • Back button doesn't work on certain overlays due to the WillPopScope not being called
